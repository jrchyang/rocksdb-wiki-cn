## 发布编号

发布版本包括三个部分：major.minor.patch ，例如 6.8.2。

自 RocksDB 3.1 版起，我们开始在 `include/rocksdb/version.h` 中定义版本宏：

```C++
#define ROCKSDB_MAJOR <major version>
#define ROCKSDB_MINOR <minor version>
#define ROCKSDB_PATCH <patch version>
```

这样，即使我们更改了你可能会用到的一些应用程序接口，你也可以让你的代码在多个版本的 RocksDB 上编译和运行。

区分主版本和次版本没有客观标准。所有功能和改进都可以归入次版本，它们适用于新选项、选项默认更改或公共应用程序接口更改（如果需要）。

当几个主要功能或改进趋于稳定时，通常会推出主版本。

补丁发布中只包含错误修复。在一个次版本的补丁发布中，不会对公共应用程序接口（API）进行任何修改，除非是为了修复一个重要的错误而不可避免地进行修改。

## 发布频率

RocksDB 有基于时间的定期发布，我们通常会尝试每个月发布一次小版本。

## 发布分支和标记

- 主分支用于代码开发，它应该在一定程度上保持稳定并处于可用状态，但不建议用于生产
- 不在主分支上发布。例如，3.0 版将有一个 3.0.fb 分支。一旦该分支足够稳定（错误修复同时进入主分支和分支），我们就会创建一个名为 "v3.0.0 "的发布标签。在主分支开发 3.1 版的同时，我们维护 "3.0.fb "分支，并在其中推送错误修复。我们在 "3.0.fb "分支上发布 3.0.1、3.0.2 等版本，并将其命名为 "v3.0.X"。一旦我们发布了 3.1，我们就会停止维护 "3.0.fb "分支。我们没有长期支持版本的概念。

下面就是一个例子：

```
-----+---------------------+-----------------------------------> branch: main
     |                     |
     |                     +--------(*)---------(*)---------------> branch 3.2.fb
     |                             3.2.0        3.2.1
     |
     +-----(*)----------(*)---------(*)---> branch: 3.1.fb
          3.1.0        3.1.1       3.1.2
```

发布分支的错误修正一般先通过主分支，然后再用 `git cherry-pick` 进行回传。(开发者注：强烈建议针对分支创建拉取请求，以便进行 CI 验证（[实例](https://github.com/facebook/rocksdb/pull/6720)），但也可从命令行推送到分支）。

例如，在 main 中修复了一个错误，需要将其反向移植到 3.1 中，顺序如下：

```
-----+---------------------+-----------------------------------[fix]-> branch: main
     |                     |                                     |  
     |                     |                                     V
     |                     +--------(*)---------(*)------------[fix]---(*)-> branch 3.2.fb
     |                             3.2.0        3.2.1            |    3.2.2
     |                                                           V
     +-----(*)----------(*)---------(*)------------------------[fix]---(*)-> branch: 3.1.fb
          3.1.0        3.1.1       3.1.2                              3.1.3
```

在正式公开发布之前，发布分支会在 Facebook 服务器中使用两周或更长时间。例如，最新的公开发布版本可能是 3.1.2，而 Facebook 正在使用来自 3.2.fb 分支的内部发布版本 3.2.1 等。我们鼓励其他人在正式公开发布之前也试用这些发布分支。当正式发布时，你可以认为它已在生产中稳定运行两周。
